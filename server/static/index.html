    ARCHITECTURE

    Three-phase training pipeline

    An LLM agent learns to be a race engineer — reading hidden state, calling the pit, managing tyres across a 12-lap sprint with six independently-scored reward dimensions.

    01

    OpenEnv Environment

    FastAPI server wrapping a deterministic F1 physics model. Each step = one strategic decision. 25+ racetrack CSVs, 6 scenario families, 5 rule-based opponents, real weather events.

    • Tyre degradation & compound rules
    • Fuel burn model per track
    • Safety car / VSC events
    • Hidden opponent strategies
    POST /reset · POST /step · GET /state
    02

    GRPO Training

    Qwen3-4B warm-started with SFT on expert traces, then GRPO-trained for 800 steps with a fully-deterministic reward function. No LLM judge — every reward is a Python check.

    • SFT warm-start → GRPO starts at 0.875
    • 800 steps · RTX 5090 · Unsloth LoRA
    • 6-dim weighted scalar reward
    • Peak 0.907 @ step 480
    train.py --backend trl --task multi
    03

    Held-out Evaluation

    Six scenario families across 5 seeds each. Trained model shows +0.32 average improvement over untrained, near-expert on weather and safety-car families.

    • Weather roulette: 0.38 → 0.95
    • Late safety car: 0.53 → 0.94
    • Dry strategy sprint: 0.51 → 0.75
    • Championship decider: 0.27 → 0.54+
    evaluate.py --model ./grpo_v1 --n-seeds 5

    Python (openenv client)
    # pip install openenv
from f1_strategist import F1Action, F1Env

with F1Env.from_env("Deltasthic/f1-strategist") as env:
    obs = await env.reset(task="weather_roulette")
    while not obs.done:
        action = my_agent(obs)
        obs = await env.step(F1Action(message=action))
    print(obs.score)  # 0.0 – 1.0
