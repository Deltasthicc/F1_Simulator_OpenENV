# results/

Canonical evaluation artefacts. **All files here are committed to git** — judges
read these to score the project.

## Contents

| File | Owner | When | What |
|---|---|---|---|
| `eval_summary.json`             | Person 2 | Phase 3 | random/untrained/trained/expert per scenario |
| `eval_curve.png`                | Person 2 | Phase 3 | Grouped bar chart, four bars per scenario |
| `training_loss_curve.png`       | Person 2 | Phase 3 | Loss + reward over training steps |
| `FINAL_RESULTS.md`              | Person 2 | Phase 3 | Narrative table + methodology notes |
| `ablation.md`                   | Person 2 | Phase 4 | Postmortem-augmented vs base trained |
| `ablation_no_memory.json`       | Person 2 | Phase 4 | Raw numbers for the ablation |
| `ablation_with_memory.json`     | Person 2 | Phase 4 | Raw numbers for the ablation |

## Generation

Everything in this folder is regenerated by:

```bash
# Eval (Phase 3)
python evaluate.py \
    --model Deltasthic/f1-strategist-qwen3-4b-grpo \
    --tasks dry_strategy_sprint weather_roulette late_safety_car championship_decider \
    --n-seeds 5 \
    --modes random untrained trained expert
# → eval_summary.json, eval_curve.png

# Loss curve (Phase 3)
python scripts/plot_training_curve.py --run-dir ./grpo_v1 --output results/training_loss_curve.png

# Ablation (Phase 4)
python evaluate.py --model Deltasthic/f1-strategist-qwen3-4b-grpo \
    --tasks ... --n-seeds 10 --no-memory --output-json results/ablation_no_memory.json
python evaluate.py --model Deltasthic/f1-strategist-qwen3-4b-grpo \
    --tasks ... --n-seeds 10 --use-memory --output-json results/ablation_with_memory.json
python scripts/diff_ablation.py results/ablation_no_memory.json results/ablation_with_memory.json --output results/ablation.md
```

## Pre-flight before submission

Verify all five required artefacts exist and are non-empty:

```bash
for f in eval_summary.json eval_curve.png training_loss_curve.png FINAL_RESULTS.md ablation.md; do
    test -s "results/$f" && echo "  ok: $f" || echo "MISSING: $f"
done
```

Anything `MISSING:` blocks the final tag. See `PRE_PUSH_CHECKLIST.md` §10.
